## 2. Phase 1：App 骨架 + 相簿

### 2.1 目標

建立 iOS/Android App，讓消費者在拍攝後能透過 App 瀏覽相簿、下載照片、進入站內溝通室。

### 2.2 開發範圍

| 模組 | 內容 |
|------|------|
| App 骨架 | React Native / Flutter 專案初始化、導航、認證 |
| 登入/註冊 | 手機/Email 登入、社群登入 |
| 我的相簿 | 拍攝相簿列表、相簿詳情 |
| 照片瀏覽 | 全螢幕瀏覽、手勢縮放、下載 |
| 站內溝通室 | 訂單溝通室（文字/圖片） |
| 推播基礎 | FCM/APNS 推播接收 |
| 再次預約 | 從 App 發起預約（導向 web 或 App 內預約） |

### 2.3 成功判準

| 指標 | 意義 |
|------|------|
| App 可用 | iOS/Android 基本功能完整 |
| 相簿體驗優於網頁 | 留存動機成立 |
| 推播可達 | 回訪召回能力 |
