# S2-015 即時媒合內部封測計畫

## 目標

在正式對外開放（S3-009）前，由 10–20 位核心攝影師與內部消費者帳號，完成即時媒合端到端驗證。

## 前置條件

| 項目 | 說明 |
|------|------|
| 後端 | `wanderlens-api` + Redis + MySQL，migrations V013–V015 已套用 |
| 種子 | 執行 `scripts/seed-match-beta.sql` |
| 功能開關 | 營運後台 `/match/monitor` 開啟即時媒合 |
| E2E | `npm run test:e2e:match`（需 API 運行） |

## 測試帳號（預設）

| 角色 | empno | 用途 |
|------|-------|------|
| 消費者 | consumer1 | 建立媒合需求、SSE 狀態 |
| 攝影師 | photographer1 ~ photographer3 | 接單、WebSocket 推播 |
| 管理員 | admin | 監控面板、功能開關 |
| 封測擴充 | beta_c01, beta_p01, beta_p02 | 內部封測批次 |

環境變數（Playwright）：

```bash
MATCH_CONSUMER_EMPNO=consumer1
MATCH_CONSUMER_PASSWORD=123456
MATCH_PROVIDER_EMPNO=photographer1
MATCH_PROVIDER_PASSWORD=123456
MATCH_ADMIN_EMPNO=admin
MATCH_ADMIN_PASSWORD=123456
API_BASE_URL=http://127.0.0.1:8080/api
```

## 封測檢查清單

### 消費者 App（S3-001）

- [ ] 首頁「即時拍攝」入口可見（功能開關 ON）
- [ ] 3 步驟流程：地點 → 確認 → SSE 等待媒合
- [ ] 媒合成功後導向付款 WebView
- [ ] 功能關閉時顯示「尚未開放」

### 攝影師 App（S3-004）

- [ ] 接單模式開關（需功能開關 ON）
- [ ] WebSocket 收到需求卡片、倒數、接受/略過
- [ ] 接受後訂單出現在訂單列表
- [ ] 首頁顯示今日/本月即時接單統計

### 營運後台（S3-008 / S3-010）

- [ ] 監控面板：搜尋中 / 已媒合數即時更新
- [ ] 30 日媒合率、fallback、逾時指標
- [ ] 功能開關可即時生效

### 預約流程（S2-006）

- [ ] 消費者 App 6 步驟預約含三池選人
- [ ] 雙機方案可選第二位攝影師
- [ ] 建單帶 `availabilityId` 鎖檔

### 行事曆（S3-005）

- [ ] 批次設定可接案時段
- [ ] 批次封鎖不可約時段
- [ ] 已預約時段顯示訂單編號

## 封測時程建議（5 工作天）

| 天 | 內容 |
|----|------|
| D1 | 環境部署、seed、E2E 自動化全綠 |
| D2 | 5 位攝影師接單演練（台北市） |
| D3 | 10 位攝影師 + 逾時降級 / fallback 觀察 |
| D4 | 功能開關 ON/OFF、推播召回交叉測試 |
| D5 | 問題彙整、Go/No-Go 決策 |

## 問題回報

記錄：需求 ID、攝影師 empno、時間戳、截圖、預期 vs 實際。優先級 P0 阻擋對外開放。

## 通過標準

- E2E 整合測試可重複通過
- 封測期間媒合成功率 ≥ 70%（有在線攝影師時）
- 無 P0 未解問題
- 營運後台數據指標可正常累積
