# wanderlens-api 開發計畫

> 本文件定義 wanderlens-api 的開發階段、範圍、優先順序與成功判準。
>
> **文件狀態**：階段規劃仍有效；完成度與缺口以 [README.md](./README.md) 與 [WanderLens_09](../WanderLens_09_待補強與缺口清單.md) 為準。最後對照：**2026-06-27**。

---

## 1. 階段總覽

| 階段 | 核心目標 | 對應 WanderLens 產品階段 |
|------|---------|------------------------|
| Phase 1 | 服務驗證 — 核心後端 API | 階段一：服務驗證 |
| Phase 2 | 服務完整化 — 多供給池 + 精修 | 階段二：服務完整化 |

## 2. Phase 1：服務驗證

### 2.1 目標

建立核心後端 API，支撐網站預約、攝影師端履約、營運後台監控、RAW 媒體管線介面。證明整個服務承諾在 API 層面成立。

### 2.2 開發範圍

#### 基礎設施層

| 模組 | 內容 | JS 參考 |
|------|------|---------|
| 專案骨架 | Spring Boot 3.x + Java 17 專案初始化 | joyshot-api 骨架 |
| 認證模組 | Spring Security + JWT（獨立密鑰）、角色權限 | JS JWT（需升級） |
| 通用回傳 | Result 統一格式、全域例外處理 | JS Result（可參考） |
| 資料庫 | MySQL 連線、MyBatis-Plus 配置、分頁插件 | JS MybatisPlusConfig |
| Redis | 連線配置、時段鎖定基礎 | 無（新增） |
| API 文件 | SpringDoc OpenAPI 3 | JS Swagger2（需替換） |
| CORS | 白名單限制 | JS CorsConfig（需收緊） |
| 日誌 | Logback 多環境配置 | JS logback（需補全 prod） |

#### 身分與權限

| 模組 | 內容 | JS 參考 |
|------|------|---------|
| User CRUD | 消費者、攝影師、造型師、營運、客服、財務 | JS User |
| 角色權限 | RBAC，角色 → 選單權限 | JS Role + Menu |
| 認證 | JWT 登入、token 刷新、登出 | JS TokenUtil（需升級） |
| 密碼 | BCrypt 加密（取代 MD5） | JS MD5Util（需替換） |

#### 預約與媒合

| 模組 | 內容 | JS 參考 |
|------|------|---------|
| ServiceType | 拍攝類型 CRUD + 多語系 | JS ServiceCat |
| Configuration | 拍攝配置（外拍/棚拍、單機/雙機、造型師） | 無（新增） |
| Availability | 攝影師行事曆 CRUD + 時段鎖定 | JS PhotographerSchedule |
| 搜尋媒合 | 依條件搜尋可用攝影師（階段一僅單池） | JS SearchingController |
| 時段鎖定 | Redis 分散式鎖，選到即確定 | 無（新增） |

#### 訂單與狀態機

| 模組 | 內容 | JS 參考 |
|------|------|---------|
| Order | 訂單 CRUD、訂單編號產生 | JS Order |
| 狀態機 | 17+ 狀態的完整狀態機 | JS 10 狀態（需擴展） |
| OrderHistory | 訂單事件紀錄 | JS OrderHistory |
| ShootEvent | 起拍/加時/結束時間戳記 | 無（新增） |
| 加時 | 加時申請 + 即時付款 | 無（新增） |
| 取消退款 | 分級取消 + 退款流程 | JS auto_cancel（需擴展） |
| 重新媒合 | 攝影師棄權自動重新媒合 | 無（新增） |

#### 金流

| 模組 | 內容 | JS 參考 |
|------|------|---------|
| ECPay 綠界 | 線上付款整合 | JS ECPayController ⭐⭐⭐ |
| 付款回呼 | 綠界回呼驗證 + 狀態更新 | JS handlePaymentSuccess |
| 加時付款 | 拍攝中即時加價 | 無（新增） |
| 退款 | 原路退款或人工匯款 | 無（新增） |
| LedgerEntry | 清算帳本（收款/抽成/應付/撥款） | JS transferred（需擴展為帳本） |

#### 站內溝通

| 模組 | 內容 | JS 參考 |
|------|------|---------|
| Conversation | 溝通室（管理/客服/訂單三通道） | 無（新增） |
| Message | 文字 + 圖片 + 系統訊息 | 無（新增） |
| 訂單通道連動 | 付款後自動開啟、結案後唯讀 | 無（新增） |
| 敏感資訊遮罩 | 手機/Email/地址自動遮罩 | 無（新增） |

#### 通知

### 2.3 需沉澱的資料

- 拍攝類型、地點、日期、價格
- 預約開始 → 付款完成 → 攝影師聯繫 → 拍攝節點 → RAW 上傳 → AI 交付的完整時間軸
- 站內溝通記錄
- 相簿瀏覽、下載、分享、公開事件
- 場景標籤（從訂單自動產生）

### 2.4 成功判準

| 指標 | 意義 |
|------|------|
| 核心API可運作 | 認證、預約、訂單、金流、溝通、通知完整跑通 |
| 訂單狀態機正確 | 17+ 狀態轉移無死鎖、無遺漏 |
| 時段鎖定可靠 | 併發預約不衝突 |
| 綠界金流正確 | 付款/回呼/退款完整 |
| 站內溝通可用 | 成交後開啟、結案後唯讀、爭議可調閱 |
| 排程任務準確 | 24h提醒、自動取消、SLA告警準時觸發 |
| 媒體介面暢通 | 與 media 服務的正確對接 |

## 3. Phase 2：服務完整化

### 3.1 目標

擴展為完整服務組合，納入攝影棚、造型師、雙攝影師、精修加購。

### 3.2 開發範圍

| 模組 | 內容 |
|------|------|
| Studio | 攝影棚供給池 CRUD + 檔期 |
| Stylist | 造型師供給池 CRUD + 檔期 + 前置緩衝計算 |
| 三池媒合 | 攝影師 ∩ 攝影棚 ∩ 造型師 檔期交集 |
| 雙機服務 | 雙攝影師配置 + 價格 + 角色分配 |
| RetouchJob | 精修工單 CRUD + 外包派工 + 成品驗收 |
| 評價系統 | 消費者評價 + 攝影師評分 + 違規紀錄 + 淘汰機制 |
| 供給治理 | 攝影師分級 + 作品集管理 + 服務範圍 + 價格帶 |
| 清算擴展 | 攝影師/攝影棚/造型師/外包費用分項帳本 |
| 溝通擴展 | 範本訊息 + 改期/加時流程整合 |

## 4. Phase 3：內容平台化

### 4.1 目標

讓相簿、公開授權、SEO 內容池、場景標籤開始產生飛輪。

### 4.2 開發範圍

| 模組 | 內容 |
|------|------|
| Album 擴展 | 拍攝歷程、年份、地點、人物關係、收藏 |
| Consent 擴展 | 多層授權（公開/作品集/行銷/商業）+ 雙向授權 + 撤回 |
| 公開內容 | 公開作品頁、地點靈感頁、攝影師作品頁 API |
| SEO 頁生成 | 結構化資料自動生成 SEO 頁 |
| 分享工具 | 分享連結、社群格式輸出、推薦連結 |
| 場景標籤擴展 | AI 影像辨識標籤 + 行為標籤 |
| 回訪召回 | 推播、拍攝週年、寶寶月份、旅遊回顧 |
| 內容治理 | 精選、審核、下架、未成年人特別處理 |

### 5.2 開發範圍

| 模組 | 內容 |
|------|------|

## 7. 風險

| 風險 | 對策 |
|------|------|
| 狀態機複雜度 | 完整測試覆蓋、狀態轉移驗證 |
| 時段鎖定競態 | Redis 分散式鎖 + 壓力測試 |
| 金流安全 | 綠界 CheckMacValue 驗證、無明文密鑰 |
| 溝通記錄量大 | 分頁查詢、歷史歸檔策略 |
| 媒體介面不穩 | 重試機制、降級策略 |
