# WanderLens UI/UX 設計規劃

> 本文件定義 WanderLens 全平台的 UI/UX 設計系統、各端點設計規範、使用者流程對應的介面設計，作為所有前端子專案的統一設計依據。
>
> 建立日期：2026-06-23

---

## 1. 設計系統總覽

### 1.1 設計原則

WanderLens 的產品橫跨「消費者面向」（web、app）與「供給/營運面向」（provider、admin、retouch），兩者的設計語言需有區別但共享基礎系統。

| 原則 | 說明 |
|------|------|
| **消費者端：情感與信任** | web/app 要讓使用者感受到專業、溫暖、可信賴，降低預約攝影的心理門檻 |
| **供給/營運端：效率與清晰** | provider/admin/retouch 要讓操作者快速完成工作，資訊密度高、操作路徑短 |
| **統一基礎** | 配色、字體、圖示、間距系統共享，各端點在此基礎上調整密度與風格 |
| **行動優先** | web 需 RWD，app 是原生行動體驗，provider 的 RAW 上傳常在現場用手機操作 |
| **多語系兼容** | 從設計階段就考慮四語系（zh/en/jp/ka）的文字長度差異 |

### 1.2 端點設計語言分類

```
┌─────────────────────────────────────┐
│          WanderLens Design System    │
│     （共享基礎：色彩/字體/間距/圖示） │
├──────────────┬──────────────────────┤
│  消費者面向    │   供給/營運面向       │
│  (B2C)        │   (B2B/Internal)     │
├──────────────┼──────────────────────┤
│  web (RWD)    │  provider (後台)     │
│  app (原生)   │  admin (後台)        │
│               │  retouch (輕量後台)  │
│  風格：溫暖    │  風格：效率           │
│  密度：寬鬆    │  密度：緊湊           │
│  互動：引導式  │  互動：直接操作       │
└──────────────┴──────────────────────┘
```

| 角色 | 色碼 | 用途 |

| **Header Logo BG** | `#000000` | provider/admin logo 區 |
| **Footer BG** | `#484E5C` | provider/admin footer |

### 2.5 色彩使用規則

1. **Primary 是唯一 CTA 色**：頁面上最重要的動作按鈕用 Primary，不稀釋
2. **Secondary 用於次要動作**：輔助性按鈕、資訊標籤
3. **Danger 僅用於破壞性操作**：刪除、下架、退款
4. **消費者端色彩飽和度可略高**：營造溫暖感
5. **後台端色彩飽和度略低**：降低視覺疲勞
6. **狀態色不與品牌色衝突**：Success 綠 vs Secondary 青藍需可區分

## 3. 字體系統

### 3.1 字體家族

```css
font-family: "Roboto", "Noto Sans TC", "Noto Sans JP", "Noto Sans KR",
             "微軟正黑體", "Microsoft JhengHei",
             "蘋果儷中黑", "Apple LiGothic Medium",
             -apple-system, BlinkMacSystemFont, sans-serif;
```

| 語系 | 字體 | 說明 |
|------|------|------|
| 中文 | Noto Sans TC / 微軟正黑體 | Google Fonts + 系統 fallback |
| 英文 | Roboto | Google Fonts |
| 日文 | Noto Sans JP | Google Fonts |
| 韓文 | Noto Sans KR | Google Fonts |

### 3.2 字體階層

| 階層 | 消費者端 | 後台端 | 行動端 | 用途 |
|------|---------|--------|--------|------|
| H1 | 32px / 700 | 24px / 700 | 28px / 700 | 頁面標題 |
| H2 | 26px / 600 | 20px / 600 | 22px / 600 | 區塊標題 |
| H3 | 20px / 600 | 18px / 600 | 18px / 600 | 卡片標題 |
| H4 | 18px / 500 | 16px / 500 | 16px / 500 | 小標題 |
| Body | 16px / 400 | 14px / 400 | 15px / 400 | 內文 |
| Body Small | 14px / 400 | 13px / 400 | 13px / 400 | 輔助內文 |
| Caption | 12px / 400 | 12px / 400 | 12px / 400 | 標籤、時間、備註 |
| Button | 16px / 500 | 14px / 500 | 16px / 500 | 按鈕文字 |

### 3.3 行高

| 用途 | line-height |
|------|-------------|
| 標題 | 1.3 |
| 內文 | 1.6 |
| 緊湊（表格/列表） | 1.4 |

### 3.4 多語系字體調整

不同語系的字寬差異需在 CSS 中調整：

| 語系 | 調整 |
|------|------|
| en | 字體可略大、letter-spacing 加 0.01em |
| jp | 字體維持、letter-spacing 加 0.02em |
| ka (韓) | 字體維持、letter-spacing 加 0.02em |
| zh | 基準值 |

CSS 實作模式（沿用 JS 精神）：
```css
/* 根元素加語系 class */
.zh-app { /* 中文基準 */ }
.en-app { font-size: +1px; letter-spacing: 0.01em; }
.jp-app { letter-spacing: 0.02em; }
.ka-app { letter-spacing: 0.02em; }
```

## 7. 各端點 UI/UX 詳細規範

### 7.1 wanderlens-web（消費者網站）

#### 首頁區塊設計

| 區塊 | 高度 | 設計要點 |
|------|------|---------|
| Banner 輪播 | 60vh | 全寬 Swiper、桌面/手機不同圖、漸層遮罩 + 標題文字 |
| 服務介紹 | auto | 淺色背景、4 大訴求卡片（超值/彈性/效率/快速） |
| 拍攝類型 | auto | Secondary 背景色、8+ 卡片網格、icon + 名稱 + 起價 |
| 預約步驟 | auto | Primary Light 背景、5-6 步驟橫向流程圖 |
| 攝影師展示 | auto | AreaSlider 縣市選擇 + PhotographerGrid 4 欄卡片 |
| 評價 | auto | Swiper 輪播、avatar + 星等 + 評論 |
| 景點 | auto | Swiper 輪播、景點照片 + 名稱 |
| IG | auto | Swiper 輪播、IG 貼文縮圖 |
| 招募 | auto | 淺灰背景、攝影師招募 CTA + FAQ 連結 |

#### 搜尋列設計

```
┌──────────────────────────────────────────────────────┐
│  [服務類型]  [日期]  [地點]  [時段]  [時數]  [人數]  [搜尋] │
└──────────────────────────────────────────────────────┘
```

- 位於 Header，點擊展開/收合
- 每個欄位為獨立卡片式選擇器（非傳統下拉）
- 服務類型：radio card（icon + 名稱）
- 日期：AirDatepicker（依語系切換 locale）
- 地點：autocomplete + Google Places（debounce 700ms）
- 時段：hour 8-22 / minute 00-50 每 10 分
- 時數：1-8 小時，每 0.5 小時一階
- 人數：大人 0-10+ / 小孩 0-10+
- 搜尋按鈕：Primary 全寬

#### 預約流程設計（Step 1-6）

```
Step 1: 拍攝類型          → 卡片選擇（icon + 名稱）
Step 2: 拍攝配置          → 自動帶入建議，可微調
         外拍/棚拍/兩者    → radio card
         攝影師人數        → 1位/2位 radio
         造型師            → 是/否 radio
Step 3: 時間與地點         → 日期選擇 + 時間選擇 + 地點輸入/攝影棚選擇
Step 4: 選擇服務人員       → 三池分別顯示（攝影師/攝影棚/造型師）
         每個清單僅顯示 3 項：作品集縮圖 + 評分 + 價格
Step 5: 確認總覽           → 所有已選項目 + 明細 + 總價
Step 6: 結帳               → 客戶資料 + 付款
```

**流程設計原則**：
- 每步一個畫面，不滾動太多
- 步驟指示器（Step 1/6）置頂
- 可返回上一步修改
- Step 2 的建議配置以「已選」狀態呈現，使用者可微調
- Step 4 的清單「選到即確定」，無需等待確認
- Step 5 的總價即時顯示，無隱藏費用

## 9. 多語系 UI 考量

### 9.1 文字長度差異

| 語系 | 相對長度 | 設計調整 |
|------|---------|---------|
| zh | 基準 (1x) | - |

| 端點 | 切換位置 | 方式 |

沿用 JS 精神，根元素加語系 class：

```css
.zh-app { /* 中文基準 */ }

.jp-app { letter-spacing: 0.02em; }
.ka-app { letter-spacing: 0.02em; }
```

## 11. 無障礙設計（Accessibility）

### 11.1 基本要求

| 項目 | 規範 |
|------|------|
| 色彩對比 | 文字與背景對比 ≥ 4.5:1（WCAG AA） |
| 焦點可見 | 鍵盤焦點需有明顯視覺指示 |
| 圖片 alt | 所有圖片需有 alt 文字 |
| 表單標籤 | 所有表單欄位需有 label |
| 語意化 HTML | 使用正確的 HTML5 語意標籤 |
| 鍵盤導航 | 所有操作可用鍵盤完成 |

### 11.2 色彩對比檢查

| 組合 | 對比 | 合規 |
|------|------|------|
| #333333 on #FFFFFF | 12.6:1 | ✅ AAA |
| #666666 on #FFFFFF | 5.7:1 | ✅ AA |
| #F37A69 on #FFFFFF | 3.1:1 | ⚠ 僅大文字 |
| #FFFFFF on #F37A69 | 3.1:1 | ⚠ 僅大文字 |
| #FFFFFF on #333333 | 12.6:1 | ✅ AAA |

> Primary 珊瑚紅對白字對比偏低，按鈕文字需用 18px+ 或 700 weight 以符合 AA。
